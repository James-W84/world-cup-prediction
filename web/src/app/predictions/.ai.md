# web/src/app/predictions/.ai.md

## 1. Local Feature Domain
- **Scope:** Group and knockout prediction UI, draft restoration, predicted standings, bracket unlocks.
- **Core Dependencies:** `useAuth`, `usePredictions`, `getMatches`, `savePrediction`, `submitPrediction`, `BracketLayout`.

## 2. Invariants & Business Logic Rules
- **State Invariants:** `KNOCKOUT_ROUNDS`: `LAST_32`, `ROUND_OF_16`, `QF`, `SF`, `FINAL`.
- **State Invariants:** Bracket unlock depends on feeder match predictions; final depends on SF feeders; SF index 2 is 3rd-place special case.
- **State Invariants:** Best third-place teams: top 8 of 12 group thirds.
- **Data Mutations:** Drafts persist under `predictions_draft`; corrupt local storage ignored.
- **Data Mutations:** Prediction save/submit must reconcile local draft, DB prediction, dirty/conflict state.
- **Edge Cases:** Standings tie-break mirrors backend bonus logic: points, wins, alphabetical.
- **Edge Cases:** Group standings use predictions only; actual match scores/goals absent.

## 3. Testing & Mocking Constraints
- **Test File Location:** Add page-level tests only after React/Next test setup exists.
- **Mocks:** Mock `getMatches` by stage; mock localStorage for draft conflict tests; freeze dates for lock displays.
