# web/src/app/leagues/.ai.md

## 1. Local Feature Domain
- **Scope:** League create/join list, league details, membership admin, join requests, leaderboard.
- **Core Dependencies:** `useAuth`, `src/lib/api.ts`, Next `Link`, `useRouter`.

## 2. Invariants & Business Logic Rules
- **State Invariants:** League list shows both member leagues and pending join requests.
- **State Invariants:** Invite codes normalized uppercase in UI before join request.
- **State Invariants:** Admin powers belong to `league.createdBy === userId`; API returns `isAdmin` and `pendingRequestCount`.
- **Data Mutations:** Create redirects to league detail; join pending stays on list with `joinStatus: pending`.
- **Data Mutations:** Approval/denial/removal/delete endpoints mutate server first, then refresh or patch UI.
- **Edge Cases:** Pending request holders can read limited league details; nonmembers without request get 403.
- **Edge Cases:** Creator cannot join own league; self-removal blocked server-side.

## 3. Testing & Mocking Constraints
- **Test File Location:** Add route tests after web test setup exists.
- **Mocks:** Mock league API functions; cover pending/member/admin branches.

## 4. Handoff Scratchpad
- **Active State:** League UI/API contract captured.
  Pending request behavior documented.
  Admin branch depends on API fields.
