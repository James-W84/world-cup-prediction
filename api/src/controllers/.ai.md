# api/src/controllers/.ai.md

## 1. Local Feature Domain
- **Scope:** HTTP handlers for auth, home dashboard, matches, predictions, leagues, admin cron triggers.
- **Core Dependencies:** `../lib/prisma`, `../utils/logger`, `../utils/helpers`, Passport session user.

## 2. Invariants & Business Logic Rules
- **State Invariants:** Always check `req.user?.id` before user-scoped reads/writes.
- **State Invariants:** Public match reads allowed; prediction, league, home routes require auth in router.
- **State Invariants:** Group predictions lock per-match after kickoff or non-`UPCOMING` status.
- **State Invariants:** Knockout bracket locks atomically at first `LAST_32` kickoff or status change.
- **State Invariants:** `DRAW` valid only for `GROUP`; knockout outcomes home/away only.
- **Data Mutations:** Respond once, then `return`; keep controller return type `Promise<void>` or `void`.
- **Data Mutations:** Log failed catch paths; preserve `{ success: false, error }` shape.
- **Data Mutations:** Prediction `submit` must verify ownership before marking submitted.
- **Edge Cases:** `matchController.getByStage` includes user prediction only when authenticated; stage currently cast with `as any`.
- **Edge Cases:** No first `LAST_32` match means knockout prediction routes treat bracket as unlocked.
- **Edge Cases:** League leaderboard tie rank changes only when lower `totalPoints`; stable secondary order by user id.

## 3. Testing & Mocking Constraints
- **Test File Location:** None existing. If added: `*.test.ts` beside controller or `api/src/controllers/__tests__/`.
- **Mocks:** Mock `prisma` methods per handler; assert status codes and response envelopes.
