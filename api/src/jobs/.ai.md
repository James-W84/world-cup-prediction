# api/src/jobs/.ai.md

## 1. Local Feature Domain
- **Scope:** Football-data match sync, completed-match scoring, group table bonus scoring, 5-minute cron.
- **Core Dependencies:** `node-cron`, `../lib/football-data`, `../lib/prisma`, `../utils/logger`.

## 2. Invariants & Business Logic Rules
- **State Invariants:** Stage points: `GROUP=1`, `LAST_32=1`, `ROUND_OF_16=2`, `QF=3`, `SF=3`, `FINAL=5`.
- **State Invariants:** Group table bonus max 4 per group; groups A-L; score only after all 6 group matches complete.
- **Data Mutations:** Completed match scoring updates prediction `isScored`, `scoredAt`, `pointsAwarded`; then increments user totals.
- **Data Mutations:** Group bonus creates `GroupTableScore` once per `userId/group`; increment user only when bonus > 0.
- **Data Mutations:** Knockout sync must not overwrite seeded bracket team names; update status/outcome only.
- **Edge Cases:** Group standings tie-break: points, wins, alphabetical; no score/GD data available.
- **Edge Cases:** football-data unavailable returns `{ created: 0, updated: 0 }`; cron must continue logging failure paths.

## 3. Testing & Mocking Constraints
- **Test File Location:** Add `*.test.ts` beside job after runner exists.
- **Mocks:** Mock football-data client, Prisma collections, and current date; never hit football-data.org in tests.

## 4. Handoff Scratchpad
- **Active State:** Scoring constants documented.
  Sync mutation guard documented.
  Cron every 5 minutes from server startup.
