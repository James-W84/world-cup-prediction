# api/.ai.md

## 1. Local Feature Domain
- **Scope:** Express API for auth, matches, predictions, leagues, scoring cron, football-data sync.
- **Core Dependencies:** `prisma`, `passport`, `express-session`, `connect-pg-simple`, `node-cron`, `football-data.org`.

## 2. Invariants & Business Logic Rules
- **State Invariants:** Authenticated user lives at `req.user`; Prisma user shape augmented in `src/types/index.ts`.
- **Data Mutations:** League join approval must update request + membership in one transaction.
- **Data Mutations:** Prediction writes use `matchPrediction.upsert` keyed by `userId_matchId`.
- **Edge Cases:** `DATABASE_URL=file:` uses BetterSQLite adapter; all other URLs use pg adapter + pg session store.
- **Edge Cases:** Config defaults conflict with README ports in places; verify callsite before changing OAuth callback/front-end URL.

## 3. Testing & Mocking Constraints
- **Test File Location:** No test harness present. Add focused tests near source only after adding runner config.
- **Mocks:** Mock Prisma singleton and football-data boundary; avoid live OAuth/cron/network in tests.
